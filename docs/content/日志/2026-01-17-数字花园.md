# 数字花园的重启：关于 anvsa 的诞生

**时间**：2026年1月17日
**坐标**：数字废墟中的一片新绿地

## 01. 缘起：逃离信息流

在这个时间节点——2026年，去搭建一个“静态网站”听起来像是一个非常复古，甚至有些笨拙的决定。
毕竟，我们有无数现成的平台：微信公众号、知乎、Twitter、小红书。它们方便、免费，并且自带流量。

但我越来越感到一种**窒息**。

在那些平台上，你是用户，更是**数字佃农**。
*   **算法绑架**：你写的东西能不能被看到，不取决于质量，取决于是否迎合了算法的喜好（情绪、冲突、热点）。
*   **时间线焦虑**：所有的内容都是“流”（Stream）。一条推文的生命周期只有几个小时。昨天写的东西，今天就被冲进了下水道。
*   **数据主权**：你的账号可能因为一句话被封禁，你的所有心血在那一瞬间清零。

我们需要一种回归。
回归到“**Web 1.0**”的精神：每个人都是一个节点，拥有自己的服务器（或仓库），拥有自己的域名。
这就是**数字花园**（Digital Garden）的概念。
这里不是“信息流”，这里是“知识库”。内容不是按时间线堆砌的，而是像植物一样，在花园里生长、连接、迭代。
这里的每一篇文章，都有它固定的 URL。不管过多少年，只要我不关站，它就在那里。

这是一种**时间主权**的体现。

## 02. 技术栈：极简主义的胜利

在技术选型上，我几乎没有犹豫就选择了 **VitePress**。

### 为什么是 VitePress？
*   **Markdown First**：写作即代码。我不需要关心数据库，不需要关心后台管理系统。我只需要打开 VS Code，写 Markdown，push 到 GitHub，一切就完成了。
*   **Vue 驱动**：作为 Vue 生态的拥趸，VitePress 让我感到亲切。它足够轻量，但也保留了强大的扩展能力（如果我想写个复杂的交互组件，随手就能写个 .vue 文件引入）。
*   **性能怪兽**：基于 Vite，毫秒级的热更新，极速的构建。对于一个以内容为主的站点，这种体验是极致的。

整个站点的结构非常清晰：
```
docs/
  ├── content/      # 所有的文章都在这里
  │   ├── 日志/
  │   ├── 杂谈/
  │   └── ...
  └── .vitepress/   # 配置文件和主题定制
```
没有复杂的 CMS，没有臃肿的依赖。纯粹，干净。

## 03. 赛博格工作流：人与 AI 的共舞

这个网站最大的特殊之处，不在于代码，而在于**它被构建的方式**。

这不完全是我写的，也不完全是 AI 写的。
这是一个典型的“**人机耦合**”（Human-AI Teaming）产物。

*   **我**（Human）：负责顶层设计、价值观输出、审美把控、以及关键的决策（比如“新奢侈品”的核心论点）。
*   **你**（AI Agent）：负责代码实现、Bug 修复、文案润色、大纲梳理、甚至是提供灵感。

在搭建这个网站的过程中，我们形成了一种奇妙的默契。
我不再是一个孤独的码农，对着屏幕发呆。我更像是一个**指挥官**，或者一个**主编**。
我发出指令：“把这个Logo改得更现代一点”、“这篇文章的语气太生硬了，要更哲学一点”。
你迅速执行，反馈结果。
我们在 VS Code 的终端里，通过一次次对话，把这个虚拟的建筑搭建了起来。

这种体验让我确信：**未来的编程，不是关于写代码（Coding），而是关于描述意图（Intention）。**
